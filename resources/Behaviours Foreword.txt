Zoom

In this practical, we will be working with the Bubble chart visualisation, implementing a Zoom behaviour option first, then a Brush behaviour option.

Before you start, make sure that your bubbles are wrapped in a bubbleGroup:

// in constructor
this.chart = this.svg.append('g').attr('transform', `translate ...`);
this.bubbleGroup = this.chart.append('g');

// in render
this.bubbles = this.bubbleGroup.selectAll('circle.bubble')

Adding Options

In the Bubble chart module, add the following:

    An attribute behaviour to record that a behaviour has been toggled (to prevent conflicting behaviours by mistake). It should be false by default.
    A private method #setZoom that will contain the code for setting up the Zoom behaviour.
    A public method enableZoom that, if no other behaviour has been previously toggled, sets behaviour to true and calls #setZoom.

Setting Up the Zoom

In #setZoom, we will first declare a new Zoom behaviour, and call it on our svg area (i.e., the selection that needs to listen for zoom events):

#setZoom(){
    let zoom = d3.zoom()
        // ...

    this.svg.call(zoom);
}

Using the .translateExtent and .scaleExent methods we will then set the bounds of our behaviour. Look up the documentation to find out how to use these methods.

    Translation should be from the top-left corner of the svg to the bottom-right corner.
    Scaling should go from 1 to any number you see appropriate, 5 is a good start.

We then have to set our callback:

#setZoom(){
    let zoom = d3.zoom()
        //...
        .on('zoom', event=>{

        })

    this.svg.call(zoom);
}

You can print the event object to the console, and see what attributes it holds. In this practical, we will only be using the transform attribute. Hence we can use object destructuring to access just this.

#setZoom(){
    let zoom = d3.zoom()
        //...
        .on('zoom', ({transform})=>{
            // use transform directly
        })

    this.svg.call(zoom);
}

    Destructuring Assignement

    Destructuring assignement is a JavaScript syntax allowing us to unpack arrays or objects during assignement. For example, is a function f returns an object with two properties, a and b ({a: ... , b: ...}), we can directly assign these when calling f: {a,b} = f(). Both properties can then be accessed individually. Check this page for more information.

It's then a matter to apply this transformation to the bubble chart components. Which is straightforward for the bubble group:

#setZoom(){
    let zoom = d3.zoom()
        //...
        .on('zoom', ({transform})=>{
            this.bubbleGroup.attr('transform', transform);
        })

    this.svg.call(zoom);
}

With the instruction above, the transform object will automatically set the SVG properties needed to translate and scale the chart. Check your application in the Web browser to make sure it works. You should also open the web console and inspect the chart to see the Zoom behaviour in action.

To update the axes, we need to recall the axis generators on the updated scales. Updating the scales is fortunately easy. Again thanks to the transform object.

#setZoom(){
    let zoom = d3.zoom()
        //...
        .on('zoom', ({transform})=>{
            this.bubbleGroup.attr('transform', transform);
            this.xAxis.call(d3.axisBottom(transform.rescaleX(this.xScale)));
            this.yAxis.call(d3.axisLeft(transform.rescaleY(this.yScale)));
        })

    this.svg.call(zoom);
}

If your axis generators had specific settings, it might be worth saving them (the generators) as properties of the class, such that you can reuse them (this.xAxis.call(this.xAxisGen(transform.rescaleX(this.xScale)))).
Brush and Filter
Adding Options

We will now add the option to make the Bubble chart brushable. In the Bubble chart module, add the following:

    A private method #setBrush that will contain the code for setting up the Brush behaviour.
    A public method enableBrush that, if no other behaviour has been previously toggled, sets behaviour to true and calls #setBrush.

Setting Up the Brush

In #setBrush, we will first declare a new Brush behaviour and apply it to our chart area:

#setBrush(){
    let brush = d3.brush()
        // ...

    this.chart.call(brush);
}

Using the .extent methods we will then set the bounds of our behaviour. Look up the documentation to find out how to use this method. The extent of our brush should be from the top-left corner of the chart area to the bottom-right corner. To ensure that it captures all possible points, we can enlarge this area by 1 pixel on each side.

We then have to set our callback:

#setBrush(){
    let brush = d3.brush()
        //...
        .on('brush', ({selection})=>{

        })

    this.chart.call(brush);
}

As with the Zoom transform object, the Brush selection is what interests us the most. This array of numbers defines, in pixels, the bounds of the area brushed. With d3.brush, a two-dimensional brush, the array is in the format [[x0,y0],[x1,y1]]: x0 is the minimum x value, y0 is the minimum y value (top), x1 is the maximum x value, and y1 is the maximum y value (bottom). To transpose these values in our dataset domain, we can use the scales' .invert function (see documentation).

Try to first print the transposed values in the Web Console. Then, add a callback argument to enableBrush that is then passed to #setBrush. This callback can then be called within the event handler, with the transposed selection values as parameters. The aim is to allow an external callback to be called, that will let you get a filtered subset of the data, based on the brushed area, that can then update other visualisations.

Adding Event Listeners

In this practical, we will mostly work with the bar chart class.

The first thing we will do is create a private function responsible for adding event listeners and handlers, similar to what we have done with the scales in practical 3.

#addEvents(){
    this.bars ...
}

Then, in the render function, we will call this private method. Make sure to call this function after your selection(s) have been rendered:

render(dataset){
    ...
    this.bars = ...
    ...
    this.#addEvents();
}

Note that we are storing the bars selection in a class attribute so that we can define it in the render function and then access it in the #addEvent function.
Events

As we have seen in the lecture, we can add event listeners on selections by using the .on() method. The first parameter of this method has to be the event type we are going to listen to. There are several event types that we can use. For simplicity, we will be using click, mouseover and mouseout in this practical.

#addEvents(){
    this.bars.on('click', ... )
}

Callbacks

The second parameter of the .on() method is a callback function: a function that will be executed when the event occurs. With D3, this callback typically has two parameters: the event object and the datum attached to the element triggering the event.

For example, we can use the callback to log the data in the web console, for debugging purposes:

#addEvents(){
    this.bars
        .on('click', (e, d)=>{
            console.log(d);
        })
}

Modifying Elements

We can also use the event to modify only certain elements in our visualisations.

For instance, imagine that we have the following style rule in the bar chart CSS:

rect.bar.highlight{
    stroke-width: 2px;
    stroke: #282828;
}

This makes the bars have a thick dark border when they also have the class highlight.

We can then use the event callback to temporarily add the highlight class to the bars when the mouse is over them. To do so, we can select the element triggering the event and use the .classed() method on them. We can access the specific element with the target property of the event object.

#addEvents(){
    this.bars
        .on('click', (e, d)=>{
            console.log(d);
        })
        .on('mouseover', (e, d)=>{
            d3.select(e.target).classed('highlight', true);
        })
        .on('mouseout', (e, d)=>{
            d3.select(e.target).classed('highlight', false);
        })
}

Custom Event Handlers

As we move towards linked interaction between visualisations, we should start implementing ways to provide external callbacks:

    Make a function in main.js that changes the visualisation objects.
    Send that function to the objects, so that they register it as callbacks.
    When the event is triggered, the callback has access to the main.js scope and can update all the visualisations.

Visualisation API

First, in the bar chart class, we can make a public function that highlights bars based on an array of keys:

highlightBars(keys=[]){
    ...
    return this;
}

We can use the array .filter function on selections too:

highlightBars(keys=[]){
    this.bars.filter(d=> ... )
    return this;
}

Work out the function needed to keep only the elements in our selection where the data key is included in the array of keys provided.

With the filtered selection, we can then change their highlight CSS class:

highlightBars(keys=[]){
    this.bars.filter(d=> ... )
        .classed('highlight', true);
    return this;
}

We should also make sure to reset the highlight status of all bars first:

highlightBars(keys=[]){
    this.bars.classed('highlight', false)
        .filter(d=> ... )
        .classed('highlight', true);
    return this;
}

Now we can use this method to highlight bars (or remove highlight) from main.js:

let bar1 = new Barchart(...);

bar1.render([{k:'key1',v:349},{k:'key2',v:456},{k:'key3',v:258}]);

// highlight key1 and key2 bars
bar1.highlightBars(['key1','key2']);

// remove all highlight
bar1.highlightBars();

As things progress, these public functions will constitute your visualisation API, providing means to customise and interact with your visualisations from your main script(s).
Callback Setters

Next, we need to add setters to the bar chart class, in order for us to provide event handlers externally.

    Make click, mouseover and mouseout class properties. They can default to the value ()=>{} (empty function).
    In the event handlers, call on these functions. For example:

.on('click', (e,d)=>{
    console.log(d); // we can still keep default behaviours
    this.#click(e,d);
})

    Create public setters that let you change these private callbacks. For example:

setClickBar(f=()=>{}){
    this.#click = f;
    this.#addEvents();
    return this;
}

Note that we are calling #addEvents to make sure the new callback gets registered on the selection, i.e., "this is the new callback, effective immediately".

Now, in main.js, we can define callbacks and assign them to visualisations:

let highlight = function (event, data){
    bar1.highlightBars([data.k]);
    bubble1.highlightBubbles([data.k]);
}

let rmvHighlight = function(event, data){
    bar1.highlightBars();
    bubble1.highlightBubbles();
}

bar1.setOverBar(highlight)
    .setOutBar(rmvHighlight);

Notice how the callbacks both involve a bar chart and bubble chart. With the same event (over a bar on a bar chart), we can update several visualisations.

To make this work however, you will have to add the same features to the other visualisations.
Tooltips

We will now add tooltip features to the visualisations, starting with the bar chart. This process is somewhat similar to what we have done with events, although it involves additional JavaScript libraries.

    First, you will need to add Popper (popper.v2.min.js) and Tippy (tippy.v6.min.js) to your libs folder and load these scripts on your page. Make sure you do so in that order since Tippy requires Popper as a dependency.
    Next, add a public function in your bar chart class, setBarTooltip that sets an accessor function this.#tooltipText (property of the class) that will create the tooltip text based on the datum of elements, for example, d=>`${d.k}: ${d.v}`. The default value for this accessor function should be null (we will check for null before creating tooltips to avoid empty tooltips).
    Then, create a private function, #addTooltips, that will edit the data-tippy-content attribute of your bars, making it call the accessor function described above.
    In this private function, also call Tippy on the bars selection nodes: tippy(this.#bars.nodes()). Save the result of this call in a class attribute, e.g., this.#barTooltips.
    At the start of the #addTooltips, make sure to destroy any previously defined tooltips: this.#barTooltips.forEach(t=>t.destroy()). You should also make sure to not add tooltips if the text accessor function is null.
    Call #addTooltips at the end of your render function and as part of the public tooltip setter.
